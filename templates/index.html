The paradigm shift from handcrafted geometry to latent optimisation. We aren't just matching pixels anymore — we're learning the structure of reality.
01
🔮

Global Feature Encoding

Twins-ViT extracts high-dimensional tokens, capturing long-range dependencies that local CNNs miss. This replaces the archaic fixed-window feature extraction.

02
🌐

All-Pairs Correlation

We compute a 4D correlation volume of all pixel pairs along the epipolar line, eliminating the "search window" constraint of legacy stereo methods.

03
🔄

Iterative Refinement

A ConvGRU "looks up" features from the correlation volume based on current dk, recursively refining: dk+1 = dk + Δd. This mimics human visual convergence.

04
🧠

Semantic Hallucination

Using learned world priors to predict depth in occluded, textureless, and specular regions — where classical geometry fails. A prerequisite for robust indoor robotics.

Iterative Update Rule:
hk = ConvGRU(hk-1, [corr(dk), dk, context])
Δd = Head(hk)
dk+1 = dk + Δd

04 — Architecture Evolution
From 3D Convolutions to Iterative Refinement
The journey from computationally expensive fixed cost volumes to elegant recurrent architectures that mimic human visual convergence.
2018 — Legacy Era

PSMNet — Pyramid Stereo Matching

3D convolutions over a fixed 4D cost volume. Computationally expensive, lacks inductive bias for out-of-distribution depth ranges. The "brute force" approach to stereo matching.

3D Conv Fixed Cost Volume SPP Module
2020 — The RAFT Revolution

RAFT-Stereo — Recurrent All-Pairs Field Transforms

Introduced iterative refinement via ConvGRU. The model "converges" on disparity like the human visual system: dk+1 = dk + Δd. Correlation lookup replaces exhaustive 3D convolution.

ConvGRU Correlation Lookup Iterative Refinement Multi-Scale

Stereo Foundation Models (CREStereo, UniMatch)

Twins-ViT for global feature encoding. Cross-attention enforces the uniqueness constraint along epipolar lines. Zero-shot generalisation across domains. Semantic hallucination fills occluded regions.

05 — Cross-Attention Visualisation
Stereo as a Sequence-to-Sequence Problem
Cross-attention enforces the uniqueness constraint — each pixel in the left image attends to its correspondence along the epipolar line in the right image.
Cross-Attention Mechanism:
Attention(QL, KR, VR) = Softmax( (QL · KRT) / √dk ) · VR

QL = queries from left image features  |  KR, VR = keys/values from right image features

Query Activation Map (Left Image)

Each cell represents a feature token. Brighter = higher activation. The dashed pink line shows the epipolar constraint.

Attention Weights (Softmax Output)

The attention "spotlight" moves along the epipolar line, finding the best correspondence in the right image.

Why Cross-Attention Matters for Stereo

Classical stereo matching uses a fixed search window along the epipolar line. Cross-attention replaces this with a learned, differentiable matching that can handle textureless regions, specularities, and occlusions. The softmax normalisation enforces a soft uniqueness constraint — each query pixel "chooses" its best match, enabling the model to hallucinate depth in regions where classical geometry fails entirely. This is the bridge from stereo matching to world understanding.
